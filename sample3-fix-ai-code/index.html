🚨 Issues Found & Fixed

🔴
SQL Injection Vulnerability
User input directly interpolated into SQL query string — allows database takeover
✅ Fixed: Switched to parameterized queries ($1, $2...)
🔴
Plain-Text Password Comparison
Passwords compared as raw strings instead of bcrypt hashes — database leak = all passwords exposed
✅ Fixed: Added bcrypt.compare() with proper password hashing
🔴
Hardcoded JWT Secret
Secret key "secret123" committed to source code — anyone with repo access can forge tokens
✅ Fixed: Moved to environment variable (process.env.JWT_SECRET)
🟡
No Rate Limiting
No protection against brute-force attacks on login endpoint
✅ Fixed: Added express-rate-limit (5 attempts per 15 minutes)
🟡
Overly Long Token Expiry
7-day token lifetime increases window for token theft exploitation
✅ Fixed: Reduced to 1 hour with refresh token pattern
