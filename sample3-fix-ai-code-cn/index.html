🚨 发现的问题及修复

🔴
SQL 注入漏洞
用户输入直接拼接到 SQL 查询字符串中 — 可导致数据库被接管
✅ 已修复：改用参数化查询（$1, $2...）
🔴
明文密码比较
密码以明文字符串比较而非 bcrypt 哈希 — 数据库泄露等于所有密码暴露
✅ 已修复：改用 bcrypt.compare() 进行安全比对
🔴
硬编码 JWT 密钥
密钥 "secret123" 直接写在代码中 — 任何有代码访问权限的人都能伪造 Token
✅ 已修复：改用环境变量（process.env.JWT_SECRET）
🟡
无频率限制
登录接口无暴力破解防护
✅ 已修复：添加 express-rate-limit（15分钟内最多5次）
🟡
Token 有效期过长
7 天 Token 有效期增加了被盗利用的时间窗口
✅ 已修复：缩短至 1 小时，配合刷新 Token 机制
